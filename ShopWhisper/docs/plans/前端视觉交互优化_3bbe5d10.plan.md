# 前端视觉与交互优化方案

## 一、视觉问题诊断

### 1.1 颜色体系混乱 — 设计 Token 未落地

项目定义了完善的 design tokens (`[src/styles/tokens.ts](frontend/src/styles/tokens.ts)`) 和 CSS 变量 (`[src/app/globals.css](frontend/src/app/globals.css)`)，但实际组件中大量使用硬编码色值，导致风格割裂：

- **登录页** (`[login/page.tsx](frontend/src/app/(auth)`/login/page.tsx)): 使用品牌色 indigo 渐变 (`#1E1B4B`, `#6366F1`)
- **管理员登录页** (`[admin-login/page.tsx](frontend/src/app/(admin)`/admin-login/page.tsx)): 使用蓝色系 (`from-gray-900 to-blue-900`, `text-blue-600`)，与品牌色不一致
- **注册页** (`[register/page.tsx](frontend/src/app/(auth)`/register/page.tsx)): 使用 `bg-gray-100` + `text-blue-600`，风格最朴素
- **Header 通知图标**: `hover:text-blue-600` 而非品牌色
- **StatCard** (`[StatCard.tsx](frontend/src/components/dashboard/StatCard.tsx)`): 全局 `accentIndex` 递增分配颜色，模块级变量在 React 渲染中不可靠
- 多个业务页面 (products, pricing, orders, content/assets) 使用 Ant Design 旧版色值 (`#1890ff`, `#f5222d`, `#ff4d4f`)

### 1.2 布局内联样式 — 未使用 Tailwind

- `[(dashboard)/layout.tsx](frontend/src/app/(dashboard)`/layout.tsx): `marginLeft: 200`, `padding: '20px 16px'`, `background: '#f3f4f6'` 全部内联
- `[Header.tsx](frontend/src/components/layout/Header.tsx)`: `style={{ padding: '0 16px', height: 64, lineHeight: '64px' }}` 内联
- `[StatCard.tsx](frontend/src/components/dashboard/StatCard.tsx)`: `bodyStyle`, `style` 大量内联

### 1.3 认证三页风格割裂


| 页面   | 背景                   | 主色     | Logo 图标              | 卡片样式                |
| ---- | -------------------- | ------ | -------------------- | ------------------- |
| 租户登录 | Indigo 渐变 + 装饰圆      | Indigo | SVG 购物车              | 圆角16, 无边框, 深阴影      |
| 管理登录 | gray-900 to blue-900 | Blue   | SettingOutlined      | 默认 Card, shadow-2xl |
| 注册   | bg-gray-100          | Blue   | ShoppingCartOutlined | 默认 Card, shadow-lg  |


三个页面应共享统一的认证页布局组件。

### 1.4 暗色模式定义但无法切换

CSS 变量已为 `data-theme="dark"` 定义了暗色值，但无切换入口，且暗色下 Ant Design 组件覆盖不完整。

## 二、交互问题诊断

### 2.1 零响应式 — 侧边栏不可折叠

- `[Sidebar.tsx](frontend/src/components/layout/Sidebar.tsx)`: 固定 `w-[200px]`，无折叠/抽屉逻辑
- `[layout.tsx](frontend/src/app/(dashboard)`/layout.tsx): `marginLeft: 200` 硬编码
- 小屏设备侧边栏遮挡内容，无任何适配
- `globals.css` 仅有 `.hide-mobile` / `.show-mobile` 两个响应式类

### 2.2 加载体验差

- 所有页面加载使用相同的 `<Spin size="large" tip="加载中..." />` 居中显示
- 项目已有 `Skeleton` 组件 (`ui/Loading/Skeleton.tsx`) 但未被任何页面引用
- 无页面切换过渡动画
- 无数据空状态引导（部分页面用 Ant Empty，部分无处理）

### 2.3 聊天页布局僵硬

- `[chat/page.tsx](frontend/src/app/(dashboard)`/chat/page.tsx): ConversationList 固定 `w-80`，RightPanel 固定宽度
- 未选择会话时无引导提示（空白区域）
- 小屏下三栏布局完全溢出

### 2.4 Settings 页面体验

- `[settings/page.tsx](frontend/src/app/(dashboard)`/settings/page.tsx): 370 行单文件，`renderContent()` 包含 5 个分支各自完整 UI
- 左侧菜单与右侧内容在移动端堆叠时缺乏分隔

### 2.5 交互反馈不足

- 部分可点击元素缺少 `cursor-pointer`
- 按钮无统一 hover/active 过渡
- 通知铃铛图标无下拉面板，点击无响应
- 表格行无 hover 高亮或点击反馈
- 操作成功/失败仅依赖 `message.success/error`，无内联反馈

## 三、优化方案 (按优先级)

### P0: 设计体系落地 — 消除硬编码色值

**目标**: 所有组件统一使用 CSS 变量 / Tailwind Token，杜绝硬编码。

- 在 `[tailwind.config.ts](frontend/tailwind.config.ts)` 中将 CSS 变量映射为 Tailwind 工具类（如 `bg-brand-500`, `text-brand-600`），确保 Tailwind 类和 CSS 变量双通道可用
- 逐文件替换所有硬编码色值:
  - `#6366F1` -> `var(--brand-500)` 或 `bg-brand-500`
  - `#1890ff` / `text-blue-600` -> `var(--primary)` 或 `text-primary`
  - `#f5222d` / `#ff4d4f` -> `var(--error)` 或 `text-error-500`
- 修复 `StatCard` 全局 `accentIndex` 问题，改为 prop 或基于 index 的确定性着色

### P1: 认证页统一 — 共享 AuthLayout

**目标**: 登录、注册、管理员登录三页风格统一。

- 创建共享的 `AuthLayout` 组件，统一:
  - 背景：品牌色渐变（与租户登录页一致）
  - 卡片样式：统一圆角、阴影、内距
  - Logo 区域：统一使用 `Logo` 组件
- 重构三个认证页面引用 `AuthLayout`
- 管理员登录页图标从 `SettingOutlined` + blue 改为品牌色系
- 注册页图标从 `ShoppingCartOutlined` + blue 改为品牌色系

### P2: 响应式侧边栏

**目标**: 支持侧边栏折叠、移动端抽屉模式。

- `Sidebar` / `AdminSidebar` 增加折叠状态 (collapsed):
  - 大屏 (>=1024px): 展开 200px
  - 中屏 (768-1024px): 折叠为 64px 仅显示图标
  - 小屏 (<768px): 隐藏，通过汉堡菜单触发 Drawer
- `layout.tsx` 的 `marginLeft` 改为响应式 Tailwind 类
- `Header` 增加移动端汉堡菜单按钮
- 使用 Zustand 或 Context 管理侧边栏状态

### P3: 加载体验升级

**目标**: 用骨架屏替代全屏 Spin，增加页面过渡。

- Dashboard 页: StatCard 骨架屏 + 趋势图骨架屏
- Chat 页: ConversationList 骨架屏
- 表格页面 (products, orders, tenants): 表格骨架屏
- Settings 页: 内容区骨架屏
- 页面切换增加 `framer-motion` 或 CSS `animate-fade-in` 过渡

### P4: 聊天页布局优化

**目标**: 自适应三栏 + 空状态引导。

- ConversationList 宽度改为可拖拽或响应式 (移动端全屏显示列表)
- 未选择会话时显示引导空状态 (图标 + 文案 + "选择一个会话开始")
- RightPanel 改为可收起面板
- 移动端: 列表 -> 聊天窗口 -> 详情 三级导航，非并排显示

### P5: 内联样式迁移到 Tailwind

**目标**: 消除 `style={{}}` 内联样式。

- `layout.tsx`: `marginLeft`, `padding`, `background` -> Tailwind 类
- `Header.tsx`: `padding`, `height`, `lineHeight` -> Tailwind 类
- `StatCard.tsx`: `bodyStyle`, `fontSize`, `fontWeight`, `color` -> Tailwind 类
- `login/page.tsx`: 渐变背景、装饰圆、卡片样式 -> Tailwind 类
- 所有 `style={{ borderRadius }}` -> `rounded-`* 类

### P6: 交互反馈增强

**目标**: 提升操作反馈的即时性和一致性。

- 统一按钮 hover/active 过渡: `transition-all duration-200`
- 可点击行/卡片增加 `cursor-pointer` + hover 背景变化
- 通知铃铛图标: 添加 Dropdown 空状态面板 ("暂无通知")
- 表格行: hover 高亮使用品牌色淡底 (`bg-brand-50`)
- 危险操作 (删除/重置) 增加二次确认的视觉强化 (红色边框弹窗)
- 表单提交增加 inline 成功/失败提示 (不仅依赖全局 message)

### P7: 暗色模式完善 (可选)

- Header 增加主题切换按钮
- 补全暗色模式下 Ant Design 组件样式覆盖
- 使用 `next-themes` 管理主题状态，支持系统偏好自动切换
